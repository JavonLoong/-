## 总体结论

这份 HTML 的**离线单文件目标基本达成**：页面没有外链脚本、外链样式、网络图片或远程请求痕迹，CSS / JS / SVG 都已内嵌在单文件中，可满足“断网打开”的硬约束（`流体力学期末复习_AI_Learning_OS.html:5-89, 92, 145-226, 4332-4382`）。

它的**图文基础也已经成型**：有总索引、公式速查、题型路径、右侧禁用条件，以及 8 个内嵌 SVG 物理示意图，整体不是纯文字长页，符合 `01_HTML整合计划.md:14-19` 和 `92_HTML视觉组件方案.md:624-633` 的大方向。

但当前版本**还不建议封板**。我认为至少有 3 类必须修正的问题：

1. 第 01 章正文被错误替换成了 `01_HTML整合计划.md`，真实章节内容没有进入 HTML。
2. 多个表格在 HTML 中已经结构损坏，出现错列，直接影响公式和条件的可读性。
3. 搜索 / 筛选 / 冲刺模式 / 补充淡化这组核心交互，只做到了“有按钮”，还没做到“行为正确”。

结论：**离线性合格，视觉基础合格，但结构和交互仍未达到可交付状态。**

## 必须修正

### 1. 第 01 章正文装配错误，当前 HTML 把整合计划塞进了 `chapter-01`

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:278-316`
- 正确源文件应为 `01_流体的物理性质.md:1-40`

现状：

- `chapter-01` 的卡头写的是“流体的物理性质”（`...html:278-280`）。
- 但展开后的正文标题却是“流体力学期末复习 HTML 整合计划”（`...html:281`），后续内容也是整合计划、调度日志、子 agent 分工（`...html:282-315`）。
- 真正的 `01_流体的物理性质.md` 内容在 HTML 中没有落进第 01 章正文。

影响：

- 这不是样式问题，而是内容装配错误。
- 学生点击第 01 章会读到项目管理信息，而不是连续介质、粘性、不可压近似等第 01 章主线。
- 同时也破坏了“14 章冲刺资料”的完整性和可信度。

审计结论：**必须修正，且应作为最高优先级。**

### 2. 多张表格在 HTML 中已发生错列，公式与条件被拆碎

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:1780`
- `流体力学期末复习_AI_Learning_OS.html:1828`
- `流体力学期末复习_AI_Learning_OS.html:2041`
- `流体力学期末复习_AI_Learning_OS.html:2515`
- `流体力学期末复习_AI_Learning_OS.html:4048`

对应源文件：

- `07_理想流体有旋与有势流动.md:27, 119-120, 433`
- `09_粘性不可压缩内部流动.md:103`
- `14_实验与综合题抢分.md:115`

现状：

- 第 07 章里 `L'=\rho U|\Gamma|`、`|\Gamma|\le 4\pi Ua` 这类含 `|` 的公式，进入 HTML 后被 Markdown 表格错误拆成多个 `<td>`。
- 第 09 章里 `` `|tau|=mu|du/dr|` `` 这类单元格同样被拆裂。
- 第 14 章里 `` `delta=|x实-x理|/x理*100%` `` 也被拆成多列。

影响：

- 表头列数与数据列数不一致，用户阅读时会把“适用条件 / 常见误用 / 典型题”错看成别列内容。
- 这类问题不是“公式排版丑一点”，而是**信息结构已经错了**，会直接误导。

根因判断：

- 相关 Markdown 源表格里有未转义的 `|`，HTML 生成阶段没有做保护。

审计结论：**必须修正。建议先修 Markdown 源或生成器的表格转义逻辑，再重新生成 HTML。**

### 3. 关键交互没有按方案真正落地：筛选不筛正文、冲刺模式反而隐藏总索引、补充淡化只淡化标签

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:72, 77, 95, 100-106, 4348-4369`
- 交互方案基线：`93_HTML交互方案.md:20-31, 41-82, 109-135, 466-469`

现状拆解：

1. **筛选只影响左侧卡片，不影响主内容区。**  
   `#filters` 的点击逻辑只对 `.chapter-card` 加 `hidden`（`...html:4359-4365`），并没有同步隐藏 `.chapter` 正文。

2. **搜索也不是统一过滤。**  
   当前搜索只扫描 `.chapter` 的 `textContent`（`...html:4348-4353`），不会统一过滤公式索引、题型路径、右侧禁用条件、自测卡片；而 `93_HTML交互方案.md:26-28` 明确要求统一过滤模块。

3. **冲刺模式把总索引隐藏了。**  
   `body.sprint #index { display:none; }`（`...html:72`），但 `93_HTML交互方案.md:115-119` 要求冲刺模式保留总索引、公式索引、题型索引、高风险条件和自测入口。

4. **“淡化补充”只淡化 badge，不淡化补充内容本体，而且默认未开启。**  
   当前 CSS 只处理 `.supplement` / `.extension` 标签本身（`...html:77`），正文仍是正常对比度；而 `93_HTML交互方案.md:128-135` 要求默认淡化，并在冲刺模式下默认隐藏补充/拓展。

影响：

- 页面现在有“按钮存在但行为不对”的假完成功能。
- 这会直接削弱“冲刺优先”和“补充内容不升格为主线”两个核心设计原则。

审计结论：**必须修正。**

## 建议修正

### 1. 自测功能只有“看答案”，距离交互方案还差一整层

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:4326-4380`
- 交互方案基线：`93_HTML交互方案.md:146-230, 470-471`

现状：

- 右侧只有 3 张自测卡。
- 行为只有“显示答案”。
- `localStorage` 只保存章节完成状态（`...html:4336-4338`），没有错题、稍后复测、限时复测 session。

判断：

- 就“有自测”和“有进度 localStorage”这两个最低要求而言，算**部分达成**。
- 但若按 `93_HTML交互方案.md` 的验收口径，它还没有形成真正可复测的练习面板。

建议：

- 增加 `掌握 / 错题 / 稍后` 三类状态。
- 至少补一个 5 分钟核心卡复测模式。
- 把错题记录和 session 结果单独存储，避免只剩章节打勾。

### 2. 可访问性语义还不够，读屏和键盘用户会吃亏

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:95, 101-106, 4327-4329, 4359-4380`
- 方案要求：`93_HTML交互方案.md:395-405`

现状：

- 搜索框没有显式 `<label>`。
- 筛选按钮没有 `aria-pressed`。
- 自测按钮没有 `aria-expanded` / `aria-controls`。
- 没有搜索结果计数的 `aria-live` 区域。
- 没有针对 `prefers-reduced-motion` 的处理。

建议：

- 保留当前 `<button>` 结构的同时，把状态语义补齐。
- 展开/折叠、自测答案显示和筛选状态要让辅助技术能读出来。

### 3. 移动端虽然能堆叠，但还不算真正“移动端友好”

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:21-22, 87-88, 98-136`
- 方案要求：`93_HTML交互方案.md:409-416`

现状：

- 窄屏时是把左侧导航整块变成页面上方长列表，而不是横向滚动筛选条或 `<select>`。
- 顶部搜索栏仍是输入框 + 两个按钮同排，超窄屏下容易显得拥挤。

判断：

- 现在的 CSS 说明“不会彻底炸版”，但还没到“手机上好用”。

建议：

- 移动端把章节筛选收成横向 pills 或下拉。
- 搜索与模式切换允许换行或分两行排布。

### 4. 搜索性能风险不高，但实现还偏粗

问题位置：

- `流体力学期末复习_AI_Learning_OS.html:4348-4353`
- 方案建议：`93_HTML交互方案.md:454-460`

现状：

- 每次输入都直接遍历所有章节 `textContent`。
- 没有 debounce，也没有预先构建 `data-search-text`。

判断：

- 以当前约 534 KB 单文件体量，桌面端大概率还能接受。
- 但低端移动端连续输入时，卡顿风险会比必要的更高。

建议：

- 增加 100-150ms debounce。
- 初始化时缓存搜索文本，避免每次重新扫整段正文。

### 5. 视觉数量够了，但有几张图还没完全守住“避免误导”

问题位置：

- HTML：`流体力学期末复习_AI_Learning_OS.html:169-174, 177-186, 189-195, 198-215`
- 视觉方案：`92_HTML视觉组件方案.md:273-310, 355-360, 515-519, 648-652`

判断：

- 当前 8 张 SVG 足够支撑“图文并茂”，这点我给通过。
- 但有几张图仍偏“速记示意”，还没完全达到方案里写的防误导要求：
  - 动量图没有把压力力画出来，和 `92_HTML视觉组件方案.md:273-274, 291-296, 650` 的要求不完全一致。
  - 管损图只画了管线和 `h_f / ζ / Σh_j`，没有体现 `Q -> A -> v -> Re -> lambda/zeta -> h_f/h_j` 的决策链，对 `V04` 的表达偏弱（`92...:306-310, 321-331`）。
  - 边界层图把外缘画成实线，且没有“0.99 约定，不是真实界面”的提示，和 `92...:359, 651` 的防误导要求有差距。
  - 可压缩图已经表达了“分段”和“激波/膨胀扇不互套”，这部分是好的。

建议：

- 不一定要加更多图，但应把关键图修到“物理含义明确且不误导”。

## 最终建议

我的建议是：**先不要封板。**

主 agent 最少应先修完下面 3 项，再做一轮快速复核：

1. 把第 01 章正文换回真正的 `01_流体的物理性质.md`。
2. 统一修复 07 / 09 / 14 章表格里的未转义 `|`，重新生成 HTML，确保表头列数与数据列数一致。
3. 把搜索 / 筛选 / 冲刺模式 / 补充淡化这组核心交互改成“行为正确”，而不是“按钮存在”。

如果时间只够做冲刺修复，优先级建议是：

1. **第 01 章错绑**
2. **错列表格**
3. **筛选与冲刺模式行为**
4. **补充内容默认降权**
5. **自测增强与可访问性补齐**

修完这三大硬伤后，这个 HTML 就会从“有完成度的初稿”变成“真的能拿去考前冲刺”的版本。
