# AI_Learning_OS 整体结构设计

> 本文档用于定义一套面向大学物理、固体力学、流体力学等课程的 AI 辅助学习操作系统。  
> 它的目标不是简单整理资料，而是把课本、PPT、作业、错题、考试题全部转化为可交互、可追踪、可复盘、可迭代的学习系统。

---

## 1. 系统定位

AI_Learning_OS 是一个以 AI 为核心的学习操作系统。

它把 AI 的能力分成多个模块：

- 资料预编译
- 知识地图生成
- 公式推导审计
- 三遍法交互学习
- 费曼复述与杠精追问
- 错题外科手术
- 错因回写讲义
- 变式题生成
- 跨章节综合题生成
- 考试模拟
- 长期复习调度

系统的核心思想是：

```text
静态资料
→ 结构化知识
→ 交互式理解
→ 推导与验证
→ 做题应用
→ 错题切片
→ 变式训练
→ 考试模拟
→ 长期记忆
```

## 3. 00_System_Init.md

### 3.1 文件定位

`00_System_Init.md` 是整个系统的总控指令。

它规定：

- AI 的教学身份
- AI 的回答风格
- AI 是否可以直接给答案
- AI 如何追问学生
- AI 如何记录学习状态
- AI 如何在不同学习模式之间切换

### 3.2 核心内容

建议包含以下部分：

```text
1. 系统目标
2. AI 角色定义
3. 学习状态机
4. 会话启动协议
5. 禁止行为
6. 标准输出格式
7. 模式切换口令
8. 学习记录规则
```

### 3.3 关键原则

AI 不应只是答案机，而应同时扮演：

- 讲义编译器
- 苏格拉底助教
- 杠精助教
- 推导审计员
- 错题外科医生
- 考试命题人
- 复习调度员

## 4. 01_Pre_Compilation 预编译模块

### 4.1 模块定位

预编译模块负责把课本、PPT、讲义、作业、考试范围等原始材料转化为结构化学习材料。

它解决的问题是：

```text
资料太散
重点不清
公式太多
题型不明
不知道先学什么后学什么
```

### 4.2 输入

可以输入：

- 课本截图
- PPT
- 老师讲义
- 作业题
- 章节目录
- 历年试题
- 手写笔记

### 4.3 输出

每一章应输出：

```text
1. 章节一句话总纲
2. 核心问题
3. 前置知识
4. 概念依赖图
5. 公式血缘表
6. 典型题型表
7. 易错点清单
8. 最低掌握标准
9. 高分掌握标准
10. 交互式 HTML 讲义草案
```

### 4.4 子文件说明

#### Prompt_HTML_Generator.md

用于把章节内容转化为单页交互式 HTML 控制面板。

应生成：

- 概念卡片
- 公式推导折叠区
- 隐藏答案题
- 分步提示按钮
- 动态图表
- 自测区
- 错题记录区

#### Component_Library.md

定义 HTML 讲义中可复用的交互组件。

包括：

- 隐藏答案组件
- 分步提示组件
- 公式推导组件
- 参数滑块组件
- 动态图像组件
- 错因记录组件
- 掌握度仪表盘

#### Prompt_Knowledge_Map.md

用于生成章节知识地图。

应输出：

```text
核心概念
子概念
依赖关系
先修知识
后续关联
考试重要度
```

#### Prompt_Formula_Lineage.md

用于生成公式血缘表。

每个公式必须说明：

```text
公式名称
公式表达
来源定律
推导前提
适用条件
不适用条件
符号含义
单位检查
常见误用
典型题
```

#### Prompt_Concept_Dependency_Graph.md

用于生成概念依赖图。

例如：

```text
牛顿第二定律
→ 动量定理
→ 角动量定理
→ 刚体转动方程
```

## 5. 02_Interactive_Internalization 交互内化模块

### 5.1 模块定位

交互内化模块负责把“看过的知识”变成“能解释、能推导、能做题的能力”。

它对应学习过程中的核心训练阶段。

### 5.2 三遍法主流程

每个知识点都必须经过三遍：

```text
第一遍：直觉理解
第二遍：公式推导
第三遍：题目应用
```

### 5.3 第一遍：直觉理解

目标：

让学生知道这个概念到底在解决什么问题。

输出应包含：

```text
这个概念解决什么问题？
如果没有它会怎样？
它在现实中对应什么现象？
它和之前哪个知识相似？
它最容易被误解成什么？
```

### 5.4 第二遍：公式推导

目标：

让学生知道公式从哪里来，而不是只背结果。

输出应包含：

```text
从哪个基本定律出发？
需要哪些假设？
每一步数学变形是什么？
中间量有什么物理意义？
最终公式什么时候能用？
什么时候不能用？
```

### 5.5 第三遍：题目应用

目标：

把知识转化为做题动作。

输出应包含：

```text
题型识别
已知量
未知量
可用公式
为什么用这个公式
标准解题步骤
检查方法
变式方向
```

### 5.6 子文件说明

#### Prompt_Three_Pass_Engine.md

三遍法执行引擎。

负责控制：

- 直觉解释
- 推导训练
- 题目应用
- 学生复述
- AI 追问
- 输出复习卡片

#### Prompt_Feynman_Debate.md

费曼复述与杠精助教模式。

学生需要先解释，AI 再从以下方向攻击：

- 概念偷换
- 公式滥用
- 条件遗漏
- 单位错误
- 边界条件不清
- 物理图像不稳

#### Prompt_Socratic_Tutor.md

苏格拉底助教模式。

AI 不直接给答案，而是通过问题引导学生自己想出来。

#### Prompt_Derivation_Auditor.md

推导审计员。

用于检查推导过程中的每一步是否合法。

#### Script_SymPy_JAX.md

符号推导与数值验证模板。

用途包括：

- 用 SymPy 检查代数推导
- 用数值代入验证公式
- 用 JAX 或 NumPy 做参数扫描
- 用图像观察公式趋势

## 6. 03_Error_Surgery 错题手术模块

### 6.1 模块定位

错题手术模块负责把错题从“我错了”变成“我知道自己为什么错，下次怎么避开”。

它是提分最快的模块。

### 6.2 错题处理流程

```text
输入错题
→ 识别真实考点
→ 复原正确路径
→ 定位错误步骤
→ 分类错误类型
→ 找到错误根因
→ 生成对抗变式
→ 回写到对应知识点
→ 加入复习调度
```

### 6.3 错误类型分类

```text
A. 概念误解
B. 公式误用
C. 条件漏看
D. 坐标系错误
E. 受力分析错误
F. 边界条件错误
G. 数学计算错误
H. 单位或量纲错误
I. 题型识别错误
J. 时间压力导致的策略错误
```

### 6.4 子文件说明

#### Prompt_Error_Slicer.md

错题逻辑切片指令。

每道错题必须拆成：

```text
题目表面信息
真正考点
所需前置概念
正确解题路径
我的错误步骤
错误类型
错误根因
下次触发条件
对抗变式题
复测时间
```

#### Prompt_Error_Classifier.md

用于对错因进行分类和统计。

输出应包含：

- 错误类别
- 严重程度
- 是否为重复错误
- 对应章节
- 对应公式
- 对应能力缺口

#### Prompt_HTML_Updater.md

让 AI 把错因写回交互式讲义。

回写格式：

```text
易错警告
错误来源
为什么容易错
错误示范
正确判断方式
一句话避坑
对抗练习
```

#### Prompt_Misconception_Detector.md

用于发现隐藏误区。

尤其适合处理：

- 看答案觉得懂
- 一换题就不会
- 公式背得出但不会选
- 概念能说但不会用

#### Error_Log_Template.md

错题日志模板。

用于长期记录所有错题和复测结果。

## 7. 04_Exam_Simulation 考试模拟模块

### 7.1 模块定位

考试模拟模块负责把知识训练转化为考试表现。

它不只是出题，而是模拟真实考试中的：

- 时间压力
- 题型混合
- 干扰条件
- 步骤给分
- 粗心错误
- 综合应用

### 7.2 模拟类型

```text
1. 章节小测
2. 半卷模拟
3. 全卷模拟
4. 限时抢分训练
5. 易错点专项轰炸
6. 压轴题拆解训练
7. 选择题陷阱训练
8. 计算题步骤分训练
```

### 7.3 子文件说明

#### Prompt_Variation_Engine.md

无限变式题库生成指令。

变式维度包括：

```text
数值变化
条件变化
已知未知互换
图像变化
多过程组合
加入干扰条件
改成证明题
改成判断题
改成实验分析题
跨章节融合
```

#### Prompt_Concept_Fusion.md

跨章节压轴大题生成指令。

用于训练：

- 多知识点缝合
- 建模能力
- 公式选择能力
- 分阶段求解能力
- 考试压轴题拆解能力

#### Prompt_Time_Attack.md

限时训练指令。

用于训练：

- 快速识别题型
- 快速选择公式
- 快速写步骤
- 在压力下减少低级错误

#### Prompt_Examiner_Mode.md

考官模式。

AI 需要按照真实考试标准评分，并指出：

- 哪一步给分
- 哪一步扣分
- 哪一步表述不规范
- 哪一步虽然结果对但过程危险

#### Exam_Report_Template.md

考试报告模板。

每次模拟后输出：

```text
总分
耗时
失分题
失分原因
最薄弱概念
最危险题型
计算错误率
公式误用率
审题错误率
下次训练任务
```

## 11. 学习状态机

每个知识点都应有状态。

```text
未接触
→ 已预编译
→ 已直觉理解
→ 已完成推导
→ 已完成基础题
→ 已完成变式题
→ 已完成错题修复
→ 已通过限时复测
→ 已进入长期复习
```

状态说明：

| 状态 | 判定标准 |
|---|---|
| 未接触 | 只知道名字，不知道内容 |
| 已预编译 | 已生成知识地图、公式表、题型表 |
| 已直觉理解 | 能用自然语言解释概念 |
| 已完成推导 | 能从基本定律推出核心公式 |
| 已完成基础题 | 能做标准模板题 |
| 已完成变式题 | 条件变化后仍能识别方法 |
| 已完成错题修复 | 已定位错误根因并通过对抗题 |
| 已通过限时复测 | 在时间压力下仍能完成 |
| 已进入长期复习 | 已纳入间隔复习系统 |

## 14. 禁止规则

AI 在本系统中禁止：

```text
1. 学生还没思考就直接给完整答案
2. 只给公式不解释适用条件
3. 只讲正确解法不分析错误解法
4. 只做单题不抽象题型
5. 只生成题不生成评分标准
6. 只讲概念不做限时训练
7. 只说“多练”这种空话
8. 不记录错因
9. 不回写知识库
10. 不安排复测
```

---

## 15. 最终能力目标

系统最终要训练出的能力是：

```text
看到题目
→ 识别章节
→ 识别题型
→ 提取条件
→ 判断适用公式
→ 建立模型
→ 推导或计算
→ 检查单位
→ 检查极限情况
→ 写出规范答案
```

而不是：

```text
看答案觉得懂
→ 换题不会
→ 考试慌
→ 错题只改答案
→ 下次继续错
```

---

## 16. 后续扩展方向

后续可以继续扩展：

```text
1. 为每个 Prompt 文件写完整提示词
2. 为每门课生成课程专用 Adapter
3. 为每章生成 HTML 交互讲义
4. 建立错题数据库
5. 建立复习仪表盘
6. 建立模拟考试报告系统
7. 让 AI 根据错题自动更新讲义
8. 让 AI 根据掌握度自动安排下一轮训练
```

---

## 17. 一句话总结

AI_Learning_OS 的本质是：

```text
把 AI 从“答案生成器”
升级为
“讲义编译器 + 推导审计员 + 互动教练 + 错题外科医生 + 考试模拟器 + 复习调度系统”。
```

它不是为了让学习看起来更复杂，而是为了让每一次学习都能留下结构化成果，并且最终转化为考试中可调用的能力。
